BASIC WORKSTATION SETUP



STEPS TO CREATE GMAIL ACCOUNT

  1. Visit Gmail: Go to the Gmail website (www.gmail.com).
  2. Click on "Create account": You'll usually find this option near the bottom of the page, but it might be at the top depending on the layout.
  3. Fill out the form: Enter your first and last name in the provided fields. Choose a username for your Gmail address. This username will be followed by "@gmail.com". If your preferred username is not available, Gmail will suggest alternatives or you can try a different one.
  4. Create a password: Choose a strong password that includes a mix of letters, numbers, and symbols. This helps secure your account. Gmail will ask you to enter the password twice to ensure you typed it correctly.
  5. Add and verify your phone number: This step is optional but highly recommended. Adding a recovery phone number makes it easier to recover your account if you forget your password. Gmail will send a verification code to the phone number you provide. Enter this code to verify your number.
  6. Add and verify your email address: Again, this step is optional but can help with account recovery. If you have another email address, you can add it and verify it.
  7. Complete the CAPTCHA: This is to confirm that you're a real person and not a bot trying to create multiple accounts.
  8. Agree to the Google Terms of Service and Privacy Policy: Read these documents and, if you agree, click on "I agree" or "Next".
  9. Your account is created: Once you've completed these steps, your Gmail account is ready to use. You can access it by signing in with your username (your Gmail address) and password.

That's it! You've successfully created your Gmail account. Now you can start sending and receiving emails.



STEPS TO CREATE GITHUB ACCOUNT

  1. Visit GitHub: Go to the GitHub website (www.github.com).
  2. Sign Up: On the GitHub homepage, you'll find a "Sign Up" button. Click on it to begin creating your account.
  3. Enter Your Information: You'll be prompted to enter the following information:
    • Username: Choose a username that will be associated with your GitHub account. If your preferred username is not available, GitHub will suggest alternatives or you can try a different one.
    • Email Address: Provide an email address that you want to associate with your GitHub account. This email address will be used for account-related notifications.
    • Password: Create a strong password for your GitHub account. GitHub will ask you to enter the password twice to ensure it's correct.
  4. Verify Your Email Address: After entering your email address, GitHub will send you a verification email. Check your email inbox for this message and click on the verification link provided to verify your email address.
  5. Complete the CAPTCHA: Confirm that you're a real person by completing the CAPTCHA verification.
  6. Choose Your Plan: GitHub offers free plans for individual users and paid plans for organizations and businesses. Select the plan that best suits your needs. For most users, the free plan (GitHub Free) is sufficient.
  7. Welcome to GitHub: Once you've completed these steps, your GitHub account is created! You can now start using GitHub to host your code repositories, collaborate with others, and contribute to open-source projects.

STEPS TO CONNECT VS CODE TO GITHUB

  1. Install Git: Before you can connect VSCode to GitHub, ensure Git is installed on your computer. You can download Git from git-scm.com and follow the installation instructions for your operating system.
  2. Install Visual Studio Code: If you haven't already, download and install Visual Studio Code from code.visualstudio.com.
  3. Open VSCode: Launch Visual Studio Code on your computer.
  4. Open or Create a Project: Open an existing project or create a new one in VSCode. The project should be stored locally on your computer.
  5. Initialize Git Repository: If your project is not already a Git repository, initialize Git by opening a terminal in VSCode (press Ctrl + on Windows/Linux or Cmd + on macOS) and run the following command in the root directory of your project:
    git init
  6. Stage and Commit Changes: Make changes to your project files and stage them for commit using Git commands in the terminal. For example:
    git add .
    git commit -m "Initial commit"
  7. Connect to GitHub:
    • Create a GitHub Repository: Go to github.com and log in to your GitHub account. Create a new repository (repo) by clicking on the "+" sign in the upper right corner and selecting "New repository".
    • Link Local Repository to GitHub Repository:
      • In VSCode, open the Command Palette (press Ctrl + Shift + P on Windows/Linux or Cmd + Shift + P on macOS).
      • Type and select "Git: Add Remote".
      • Enter the URL of your GitHub repository (e.g., https://github.com/username/repository.git). Replace username with your GitHub username and repository with the name of your repository.
      • Press Enter to confirm.
  8. Push to GitHub: After linking your local repository to the GitHub repository, push your changes from VSCode to GitHub:
    • Open the Command Palette again (Ctrl + Shift + P on Windows/Linux or Cmd + Shift + P on macOS).
    • Type and select "Git: Push".
    • Choose the remote repository (GitHub) where you want to push your changes.
    • Enter your GitHub credentials if prompted.
  9. Verify on GitHub: Go to your GitHub repository in a web browser to verify that your changes have been pushed successfully.

By following these steps, you can effectively connect Visual Studio Code to GitHub, enabling seamless version control and collaboration for your projects.



STEPS FOR CLONING REPOSITORIES

  1. Find the Repository on GitHub:
    • Go to github.com and log in to your GitHub account.
    • Navigate to the repository you want to clone. You can search for it or access it directly via the URL.
  2. Copy the Repository URL:
    • On the repository page, click on the green "Code" button. This will show you a dropdown with the repository's URL.
    • Copy the HTTPS or SSH URL (depending on how you want to authenticate and access the repository).
  3. Launch Visual Studio Code on your computer.
  4. Press Ctrl + Shift + P (Windows/Linux) or Cmd + Shift + P (macOS) to open the Command Palette.
  5. Clone Repository:
    • Type and select "Git: Clone" from the Command Palette.
    • Paste the copied repository URL into the input box and press Enter.
    • Choose a local directory where you want to clone the repository.
  6. After cloning completes, VSCode will ask if you want to open the cloned repository. Select "Open".

By following these steps, you can effectively clone a repository from GitHub to your local machine using Visual Studio Code.



STEPS TO PUSH CHANGES TO GITHUB:

  1. In VSCode, make changes to the files within the cloned repository.
  2. Stage Changes:
    • Open the Source Control view in VSCode by clicking on the Source Control icon in the Activity Bar on the side (or use Ctrl + Shift + G).
    • Review the changes to your files. Click on the + button next to each file or use the + button at the top to stage all changes.
  3. Commit Changes:
    • Enter a commit message that describes your changes in the text box at the top of the Source Control view.
    • Press Ctrl + Enter (Windows/Linux) or Cmd + Enter (macOS) to commit the changes.
  4. Push Changes to GitHub:
    • After committing your changes, click on the ellipsis (...) in the Source Control view and select "Push".
    • Choose the branch you want to push to GitHub and confirm by clicking "OK".
  5. Go to your repository on GitHub in a web browser to verify that your changes have been pushed successfully.

By following these steps, you can effectively make changes, and push those changes back to GitHub.



STEPS TO PUBLISH REPO AS LIVE SITE

  1. Prepare Your Repository:
    Ensure your repository contains all necessary files and configurations for your static website:
    • HTML, CSS, JavaScript: Include all HTML pages, CSS stylesheets, JavaScript files, and any other assets (images, fonts, etc.) required for your site.
    • Branch: Make sure your repository is on the branch (often main or master) that you want to deploy from.
  2. Enable GitHub Pages:
    GitHub Pages allows you to host static websites directly from your GitHub repository. Here’s how to enable it:
    1. Navigate to Repository Settings:
      • Go to your GitHub repository in a web browser.
      • Click on the "Settings" tab located near the top-right of the repository.
    2. In the Settings page, scroll down to the "GitHub Pages" section.
    3. Under "Source", you'll see a dropdown menu. Select the branch you want to deploy from (e.g., main or master).
    4. GitHub Pages will automatically save your selection. It will also provide you with the URL where your site will be published (e.g., https://username.github.io/repository).
  3. Verify and Deploy:
    After enabling GitHub Pages and configuring your domain (if applicable), GitHub will automatically build and deploy your site. It may take a few minutes for the initial deployment to complete.
    • Once deployed, access your live site using the URL provided in the GitHub Pages section of your repository settings (or your custom domain if configured).
    • Whenever you push changes to the selected branch of your repository, GitHub Pages will automatically rebuild and update your site. You can verify your changes by revisiting the site URL.

By following these steps, you can easily publish your repository as a live site using GitHub Pages, making your static website accessible to anyone on the internet.
